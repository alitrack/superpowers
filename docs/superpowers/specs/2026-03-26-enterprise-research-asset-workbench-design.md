# 企业研究资产工作台设计

## 摘要

这个项目不应该被定义成“另一个通用 AI 工作空间”。它第一阶段最可信、最有价值的切入口，是：

**面向企业战略 / 战投团队的研究资产工作台。**

产品外层可以借鉴 Flowith 这类 AI 工作空间的吸引力，例如：

- 工作空间而不是聊天框
- 多线程探索而不是线性对话
- 知识、探索、执行放在同一个环境里
- 有 Agent 在后台持续推进任务

但真正的产品壁垒，不应该来自“更像 Flowith”，而应该来自：

**把研究过程沉淀成可复用的组织资产。**

一句话定义：

> 这是一款漂亮的 AI 原生研究工作空间，但它最核心的价值不是更快给答案，而是把战略问题如何被定义、拆解、验证、判断和收敛，沉淀成团队可以继承的研究资产。

## 为什么是这个方向

### 为什么不应该做成泛化 AI 工作空间

通用 AI 工作空间通常会在广度上竞争：

- 更多模型
- 更多 Agent
- 更强并发
- 更大的上下文
- 更丰富的通用创作能力

这些点有演示吸引力，但不一定能在高价值企业团队内部形成长期留存。

企业战略 / 战投团队最痛的问题，不是“没有 AI 可用”，而是：

- 同类研究不断重复做
- 关键假设没有被结构化保留
- 证据散落在聊天、文档、PPT、飞书和个人脑子里
- 为什么会得出某个结论无法被完整继承
- 换人之后，研究理解和判断链条会明显断层

所以第一阶段最应该解决的，不是“更快生成结论”，而是：

**让一次性研究劳动变成可复用的团队资产。**

### 为什么这条路更值得投

从投资人视角，这条路比“讲一个大而全的 AI workspace 故事”更扎实，因为它同时具备：

- 清晰的高价值客户
- 足够深的专业痛点
- 组织级复利
- 从单点产品长成平台的明确扩张路径

这个产品真正改变的，不是某一次输出的速度，而是企业战略研究的生产方式。一旦团队开始积累：

- 问题树
- 假设结构
- 证据链
- 判断路径
- 结论演化
- 可复用研究模板

迁移成本和留存都会自然增强。

## 与 Flowith 的关系

### 应该借鉴什么

Flowith 的公开定位说明了一件事：市场已经开始接受“AI 工作空间不等于聊天框”。

从其公开主页、文档和产品描述中，可以提炼出这些值得借鉴的点：

- `workspace` 心智，而不是单轮问答
- `canvas` / `context playground` 这类多线程工作界面
- `knowledge + execution` 放在同一环境中
- Agent 在后台推进任务，而不只是回答问题

可参考的公开材料：

- https://flowith.io/
- https://doc.flowith.io/
- https://doc.flowith.io/other-questions/subscriptions-and-billing
- https://flowith.io/blog/knowledge-garden/

因此，真正该借的是：

- **工作空间心智**
- **多线程探索结构**
- **知识与任务联动**
- **Agent 作为工作单元**

### 不应该照抄什么

第一版不应该照抄 Flowith 的大盘子产品形态：

- 不做泛创作平台
- 不做通用 Agent 市场
- 不做“模型数 / 并发数 / 套餐档位”主导的商业表达
- 不做弱语义、强自由度的通用白板

因为这些东西会让产品失焦，最后变成“看起来很强，但没有一条价值链真正做深”。

### 应该如何定义差异

Flowith 可以被视为“通用 AI 工作空间方向成立”的市场教育者。

而这个项目的正确定位应该是：

> 一个专门为企业战略 / 战投团队设计的研究资产工作台。

差异不在广度，而在深度：

- 更懂研究问题如何拆解
- 更懂证据和判断如何连接
- 更懂结论为什么会成立
- 更懂研究过程如何沉淀成团队资产

## 产品定位

### 产品类别

建议把产品类别固定为：

**企业研究资产工作台**

而不是：

- 头脑风暴工具
- AI 报告生成器
- 通用 AI 画布
- 带 Agent 的知识库

### 第一批客户

最可信的第一批客户是：

- 企业战略团队
- CEO 办公室 / 战略规划部门
- 战略投资 / 战投 / 创新战略团队

这些团队比泛知识工作者更适合作为切口，因为他们：

- 单次研究价值高
- 问题重复率高
- 研究结果影响真实组织决策
- 最容易从“研究资产沉淀”中获得长期价值

### 核心价值主张

核心主张建议固定为：

**让战略研究过程可见、可复用、可累积。**

更展开一点，可以表达成：

- 让问题拆解不再丢失
- 让证据链不再散落
- 让判断过程可以复盘
- 让同类项目不再从零开始
- 让研究结论成为组织资产，而不只是个人输出

## 产品原则

第一版建议遵循以下原则。

### 1. 外层要漂亮，内层要严格

界面应该让用户一眼感到“这不是普通 chat”，但内部结构必须保持强语义和强边界，不能沦为纯展示型白板。

### 2. 先帮助定义问题，再帮助形成答案

战略研究里最贵的错误，常常来自问题定义错位，而不是材料不够多。所以产品必须先强在问题建模和假设树，而不是急于给结论。

### 3. 证据不等于判断

材料、证据、阶段性判断、最终结论必须是不同对象。否则最后只会得到信息堆积，而不是研究资产。

### 4. 分支必须被保留

高质量研究不是一路线性推进，而是不断比较、排除、修正。被放弃的分支也是资产，而不是噪音。

### 5. 结果应该是“过程资产”，不只是文档

一轮研究结束后，留下来的不应该只是一个结论页或报告，而应该是一套可以复用的研究结构。

## 第一版产品结构

第一版建议采用三层结构。

### 第一层：主画布，负责呈现研究过程

主画布不是通用白板，而是只承载 5 类核心对象：

- 研究问题
- 子问题 / 假设
- 证据
- 判断
- 结论 / 下一步

这 5 类已经足够让产品具备工作空间吸引力，同时又不失去研究语义。

### 第二层：右侧检视器，负责理解单个资产

点击任意节点，右侧检视器展示：

- 这个节点是什么
- 它依赖什么
- 它支持或反驳哪个判断
- 当前状态 / 可信度 / 未解问题
- 推荐下一步动作

这层决定了产品是不是“真正可工作的工具”，而不是一张好看的图。

### 第三层：持续收敛的结果区

结果区不应该等最后才出现，而应该在研究过程中持续显示：

- 当前最佳结论
- 当前最强理由
- 当前未解决风险
- 当前下一步建议

这样用户会感觉自己始终在逼近答案，而不是陷入无限发散。

## 第一版核心工作流

第一版建议围绕一条非常清晰的工作流来设计。

### 1. 从一个高价值问题开始

例如：

- 是否进入某个市场
- 是否投资某家公司 / 某个赛道
- 某个战略动作是否值得推进

### 2. 把问题拆成假设树

系统帮助用户形成 3 到 5 个关键研究分支，例如：

- 市场空间是否足够大
- 时间窗口是否成立
- 壁垒是否真实存在
- 执行成本是否可控
- 失败风险来自哪里

### 3. 让分支并行推进

每个分支可以继续生长出：

- 待验证问题
- 证据收集任务
- 支持与反驳材料
- 中间判断

### 4. 把证据提升为判断

系统帮助用户明确：

- 哪条证据支持什么结论
- 哪条证据反驳什么假设
- 哪个判断仍是临时性的

### 5. 收敛成结论，同时保留备选路径

研究结果需要清楚显示：

- 当前最佳结论
- 支撑它的最强理由
- 被放弃但仍值得记住的备选路径
- 尚未解决的关键风险

### 6. 将整轮研究保存为团队资产

每轮结束后，至少沉淀出：

- 研究问题
- 假设树
- 证据图谱
- 判断路径
- 最终建议
- 下一步行动

## 第一版必须做强的能力

第一版必须真正做强下面这些能力：

- 研究问题建模
- 子问题 / 假设树拆解
- 多分支研究推进
- 证据与判断分离
- 持续收敛的结论区
- 结果沉淀为团队资产
- Agent 辅助推进分支、补证据、整理判断和后续动作

## 第一版必须刻意不做的能力

第一版必须刻意克制，避免失焦。

- 不做通用白板
- 不做复杂多人协作和 Presence
- 不做全企业知识中台
- 不做重型跨系统执行编排
- 不做泛行业全覆盖
- 不做以模型 / 套餐 / Agent 市场为主的商业包装

这些以后都可能要做，但不应该进入第一阶段。

## 产品壁垒

这个产品的壁垒建议分成四层来理解。

### 1. 工作流壁垒

它深度嵌入战略研究工作流，而不是泛用 AI 使用场景。

### 2. 资产复利壁垒

每一轮研究都会沉淀为可复用资产，而不是一次性答案。

### 3. 组织记忆壁垒

团队不再只保留结果，而是保留：

- 问题如何定义
- 哪些假设被提出
- 哪些证据支持或反驳
- 为什么最终选了这个方向

### 4. 界面心智壁垒

大画布和工作空间感可以帮助产品在感知层拉开差异，但它必须服务于更深的研究资产壁垒，而不能取代它。

## 投资人叙事

最强的投资人叙事建议是：

> 这不是一个更漂亮的 AI 助手，而是一套把战略研究过程转化为组织资产的系统。

更完整的增长路径可以这样讲：

1. 先用一个窄而深的研究资产工作台打透企业战略 / 战投团队
2. 再扩到其他高价值研究团队，如咨询、投研、产业研究、政策分析
3. 最后把资产图谱、Agent 推进层、权限、模板和记忆系统平台化，长成更广义的企业 Agent Workspace

## 专业专家叙事

最强的专家侧叙事建议是：

> 这个产品不是替你下判断，而是把判断如何形成沉淀下来。

专家愿意长期留下来的前提，是产品真正做到：

- 帮助把问题建对
- 让证据和判断严格区分
- 保留被放弃的路径
- 支持下一轮研究直接复用
- 保持专家对最终解释权和判断权的主导

## 推荐的一句话定位

建议使用下面这句作为正式定位：

> 企业战略 / 战投团队的研究资产工作台。

如果需要更完整的一句产品介绍，可以使用：

> 一款漂亮的 AI 原生研究工作空间，帮助企业战略 / 战投团队把研究过程沉淀成可复用的组织资产。
