## Why

当前 `brainstorm-server` 虽然已经具备真实 Codex backend、skill bootstrap、单入口和更像脑暴的 summary，但整体仍然是“功能切片的叠加”，不是用户可以直接验收的完整 brainstorming 成品。用户现在无法确信：自己抛出的议题是否真的被 Codex 按当前 `brainstorming` skill 推进，系统提出的问题是否来自真实 skill-guided reasoning，最终结果是否已经达到一个成熟的脑暴完成品，而不是另一个半成品 summary。

现在需要把目标从“逐步修补一个可演示原型”切换成“交付一个可验收的 finished product”：用户给出脑暴问题，系统围绕该问题推进 brainstorming，并最终稳定地产出一个成熟、可回看、可审计来源的完成品。

## What Changes

- Define an end-to-end browser brainstorming completion flow that starts from the user's seed and does not mark the session complete until a mature brainstorming deliverable is produced.
- Add a deliverable quality gate so `summary` / `artifact_ready` only appear after the session has produced a complete brainstorming result rather than a shallow selection recap.
- Introduce first-class provenance for visible questions and final artifacts so developers can verify whether a question/result was generated by real Codex runtime under the current repository skills or by a fallback path.
- Add developer-facing inspection surfaces or API responses for provenance while keeping the normal user-facing UI focused on the brainstorming artifact, not CLI/protocol details.
- Add acceptance-oriented quality fixtures and execution gates so implementation is tracked against a "finished brainstorming product" bar rather than isolated partial behaviors.

## Capabilities

### New Capabilities
- `brainstorm-finished-deliverable`: Covers the mature brainstorming output contract and the completion gate that determines when a session is truly done.
- `brainstorm-generation-provenance`: Covers traceability for visible questions and final deliverables, including whether the real skill-guided Codex path or a fallback path generated them.

### Modified Capabilities
- `structured-brainstorming-runtime`: Change runtime completion behavior from lightweight handoff generation to finished-deliverable orchestration with completion gating.
- `structured-brainstorming-flow`: Change browser session flow so the visible experience is organized around reaching a finished brainstorming artifact, not merely stepping through questions.
- `structured-brainstorming-messages`: Change completion-message expectations so summary/artifact completion preserves a mature brainstorming result rather than only a selected-path recap.

## Impact

- Affects `skills/brainstorming/scripts/codex-runtime-adapter.cjs`, `web-session-manager.cjs`, `server.cjs`, `web-app-shell.html`, and possibly `structured-host.cjs`.
- Adds persisted session state for deliverable completeness and provenance.
- Requires new or expanded tests for artifact quality, provenance traceability, and end-to-end completion gating.
